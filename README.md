# 囧氏自釀宮保拼音縮略碼

配方： ℞ **lotem/rime-combo-pinyin-briefs**

Rime [宮保拼音](https://github.com/rime/rime-combo-pinyin) 縮略碼

## 簡介

宮保拼音是由居戎氏設計、在標準電腦鍵盤上多鍵並擊輸入拼音的方法。

*並擊* （chord-typing）是指：同時按下鍵盤上的多個按鍵。

### 學習資料

  - [宮保拼音教程](https://github.com/rime/home/wiki/ComboPinyin)
  - [宮保拼音鍵位練習](https://lotem.github.io/typewriter/)

  鍵位練習軟件中，有「縮略碼」練習，舉了幾個例子演示縮略碼的用法。
  支持回車輸入自定義的縮略碼，格式爲：宮保並擊碼 `=<` 一組拼音 `>`，例如 `SHGUA=<shu ru fa>`。

回車後，將 drill.txt 文件內容粘貼進去，可體驗下文「縮略法」中的示例。

### 縮略碼

七指禪佈局可支持用戶自定義的縮略碼，利用閒置的並擊組合快速輸入常用詞。

縮略碼不遵循嚴格的編碼規則，需要使用者根據自己的用詞習慣和對拼音的理解逐一制定。
可將詞語中標誌性的語音信息壓縮在一次並擊的編碼中以方便記憶，但不得與已有並擊碼衝突。

[℞ `lotem/rime-combo-pinyin-briefs`](https://github.com/lotem/rime-combo-pinyin-briefs) 可作參考。

#### 筆者制定縮略碼的原則

縮略碼爲有損編碼。不具有普遍性。
以常用雙音節詞爲縮略的主要目標，外加少量極高頻三音節詞。

制定縮略碼的方法是將取詞中每個字的特徵音素，選取提示性強、指法合理的組合，整合成一個並擊碼，且須規避基本音節和已有縮略碼。

筆者有特別的輸入習慣：並擊碼按照方音區分尖團。這樣一來，基本音節要佔用更多的並擊碼。
用家如果無需輸入尖音，可以利用表示尖音的並擊碼制定縮略碼。例如尖音「西」的並擊碼爲 `SI`，如果用作縮略碼，可表示「所以」。

保持標準指法中各手指的分工，不挪用、借用。如：「困難」的聲母編碼爲 `K, LD`，其中 `K, D` 均由左手 2 指負責，
如果要並擊這兩個聲母，勢必借用其他手指、打亂標準指法。因此捨棄這種縮略方法。

#### 技術實現

目前我只採用將並擊碼映射到拼音音節序列的實現方法。
在將並擊碼轉換成拼音音節的規則之前，插入一組規則，其中每條規則將一個具體的並擊碼轉換成對應縮略詞的拼音序列。
這樣做的優勢是簡單易行，不需要修改拼音詞典。劣勢是縮略碼不能直接定位字詞，仍需選擇重碼字。

#### 何時制定縮略碼

不應匆匆制定大批縮略碼。
日常並擊輸入時，留意到經常重複輸入的多音節詞，經過反覆練習，仍嫌指法不夠利落，應當制定縮略碼。
例如，我制定的第一個縮略碼是「什麼」這個絕對高頻的雙音節詞。
因爲左手要做兩次二指並擊，且不在同一行上，不論多麼熟練，都無法達到理想中流暢的速度。

### 縮略法

下文介紹筆者常用的縮略法。願拋磚引玉。
用家應根據自己的實際情況設計縮略法。

#### 切音法

「切音法」是最自然、提示性最強的縮略方法，主要針對雙音節詞，二字的聲母、韻母各取其一，任一部分都可以帶上介音，組合成一個形如音節的編碼。

舉例：

 - `FAO` 略代「符號」，取前字的聲母、後字的韻母。
 - `HGIA` 略代「人家」，恰如洛陽方言中的合音詞。
   可見切音法是符合語言規律、有現實基礎的。
 - `KIAR` 略代「起來」，這也是個合音詞。
   要說明的是：`KIAR`    也是基本音節`KIAO`「喬」的替代指法。
   我決定只使用標準編碼 `KIAO`，而將並擊碼 `KIAR` 改做縮略碼。
 - `LIAR` 略代「厲害」。
 - `CUARO` 略代「情況」。前字取尖音聲母 `C`，但不取前字的介音。
 - `ZFIANE` 略代「這樣」。

「切音法」的缺點是編碼空間有限，容易與基本音節碼撞車。

#### 雙聲疊韻略法

「雙聲、疊韻」略法，手寫速記常設有，可省略二字詞中重複出現的聲母或韻母。

聲母重複者，將兩個韻母鍵位合併。

  - `GÜAE` 略代「感覺」，聲母鍵 `G` 重複，從前字中取元音 `A` 爲特徵，後字取韻母 `ÜE`。

從方音中取碼，有助於創造獨特的並擊組合。

  - `GUAE` 略代「國家」，聲母鍵 `G` 重複，前字取方言中的韻母 `UE`，後字取主要元音 `A`。
  - `GIARE` 略代「解決」，聲母鍵 `G` 重複，前字取主要元音 `E`，後字取方言中的韻母 `IAR`。

疊字，或是聲韻皆相近者，省略後的編碼恰是基本音節或不符合並擊指法，可以用相鄰的鍵表示聲母重複。

  - `SHIE` 略代「謝謝」，聲母 `S` 重複。
  - `SHÜAN` 略代「喜歡」，聲母鍵 `H` 重複，同時合併介音。

聲母不同而鍵位重複，也可仿此用增加相鄰鍵的方法表示。

  -  `CLDI` 略代「哪裏」，兩個聲母 `LD, L` 有重複的鍵，於是增加鄰鍵表示重複，略作 `CLD`。

#### 雜燴略法

聲母、韻母均不同，但能夠合併爲一次並擊，也可仿雙聲、疊韻法制定縮略碼。
這類縮略碼編碼空間廣闊，易於構建獨特的並擊組合，因此爲數衆多。

舉例：

  - `ZGI` 略代「自己」。
  - `CGIA` 略代「參加」。
  - `LDUARO` 略代「勞動」。

前後字不限取碼順序，合成後一律按照並擊鍵序排列。

  - `HGIE` 略代「結合」。
  - `ZHAR` 略代「現在」。前字取聲母鍵 `H`，後字全取。
  - `SBÜ` 略代「必須」。
  - `SLIANE` 略代「理想」。
  - `SHURE` 略代「社會」。

聲母 `ZF, CL, SH` 可省略爲單鍵 `Z, C, S`。

  - `SDAN` 略代「但是」
  - `SKAR` 略代「開始」
  - `SGIE` 略代「世界」
  - `SGIAN` 略代「時間」
  - `CHIAN` 略代「出現」
  - `ZFBAR` 略代「百分之」

借用方音制定簡潔、順手的縮略碼。

  - `FIANE` 略代「非常」。
    前字取方音⟨fi⟩的編碼 `FI`，可避免 `R` 與 `N` 鍵的指法衝突，同時保留有效的語音提示。
  - `SHÜE` 略代「學習」。尖團字聲母分別取 `H, S`。
  - `SHIR` 略代「所有」。前字聲母取 `SH`。

部分韻母可省略爲單鍵。

`O` 略代⟨ao⟩⟨ou⟩⟨uo⟩等韻母及詞尾「個」。

  - `SIO` 略代「需要」。
  - `ZFO` 略代「知道」。這是中原官話的合音詞。
  - `SHO` 略代「時候」。
  - `ZGO` 略代「中國」。
  - `ZGN` 略代「中國人」。
  - `NO` 略代「我們」。
  - `GINO` 略代「經過」。
    用 `O` 代替 `INE` 中的 `E`，並提示後字的韻母。
  - `ZO` 略代「這個」。
  - `ÜO` 略代「一個」。
  - `GIO` 略代「幾個」。

### 融合介音

爲了與基本音節區別開，可將介音融合或通借爲另一介音。

常見的變化有：

`I + U = Ü`，如：

 - `SHGÜ` 略代「技術」
 - `HGÜAN` 略代「軟件」
 - `TIRO` 略代「提供」
 - `TÜN` 略代「問題」
 - `ZFÜ` 略代「主義」

開口呼、合口呼介音互相通借。

 - `HGO` 略代「如果」

齊齒呼、撮口呼介音互相通借。

 - `ÜO` 略代「一個」

#### 介音略代聲母

如兩個聲母指法衝突，無法並擊，可添加介音略代其一。

用介音 `U` 提示省略的脣音聲母。

  - `BUA` 略代「辦法」。`U` 提示聲母⟨f⟩。
  - `DUA` 略代「代碼」。`U` 提示聲母⟨m⟩。
  - `FBUAN` 略代「麻煩」。`U` 提示聲母⟨f⟩。
  - `HUANE` 略代「方向」。`U` 提示聲母⟨f⟩。
  - `FUARO` 略代「方法」。
    `U` 提示後字的聲母⟨f⟩，同時與前字的韻母 `ANE` 結合成 `UARO`。

用介音 `I` 提示省略的捲舌聲母。

  - `FIAN` 略代「發展」。`I` 提示聲母⟨zh⟩。
  - `HGIAN` 略代「仍然」。`I` 提示聲母⟨r⟩。

將原有的介音 `I`、`U` 與提示聲母的介音融合爲新的介音。

  - `TÜE` 略代「特別」。
    前字取聲母 `T`；後字的聲母⟨b〉取合口特徵，再與韻母 `IE` 融合爲 `ÜE`。

#### 變形略法

基本音節中，舌尖元音 `R` 通常省略，可在並擊組合中省略聲母、用 `R` 表示韻母。

  - `ZR` 略代「這是」。`R` 提示後字韻母爲舌尖元音。
  - `LARO` 略代「老師」。`R` 提示後字韻母爲舌尖元音。
  - `SGR` 略代「公司」。`R` 提示前字韻母爲舌尖元音。

除舌尖元音外，也可用其他聲母、韻母的替代形式編製縮略碼。

  - `CHAN` 略代「生產」。區別於基本音節 `CLAN`。
  - `CHUAR` 略代「出來」。區別於基本音節 `CLUAR`。
  - `ZHAN` 略代「戰士、展示」。區別於基本音節 `ZFAN`。
  - `ZHUN` 略代「中文」。區別於基本音節 `ZFUN`。
  - `ZHR` 略代「只是、知識」。區別縮略碼 `ZR`「這是」。

  - `SURE` 略代「所謂」。`UR` 改爲其「完全」形式 `URE`。
  - `HIAE` 略代「一下」。用 `AE` 代替 `A` 以區別於基本音節 `HIA`。
  - `ÜANE` 略代「永遠」。
     ⟨iong⟩的替代形式 `ÜNE` 與 `ÜAN` 有重合的鍵位，可組合成指法便利的並擊碼。

#### 鏡像借鍵法

如果某些讀音重複或並擊指法困難，可借用對側鏡像位置的鍵位代替。

左側的 `H` 鍵借作 `N`。

  - `HPAN` 略代「鍵盤」。`H` 略代前字中的 `N`。
  - `HGIR` 略代「研究」。`H` 略代前字中的 `N`。
  - `HKUAN` 略代「困難」。`H` 略代前字中的 `N`。
  - `CHÜAN` 略代「完全」。`H` 略代前字中的 `N`。

左側並擊碼 `HG` 或 `LD` 借作 `IN`。

  - `HGINE` 略代「引擎」。`HG` 略代前字韻母 `IN`。
  - `LDUR` 略代「因爲」。`LD` 略代前字韻母 `IN`。
    由於鏡像所得 `HGUR` 是一個基本音節，改用上排的 `LD`。

左側並擊碼 `SHG` 借作 `INE`。

  - `SHGAR` 略代「應該」。`SHG` 借作前字韻母 `INE`。
  - `SHGUAN` 略代「環境」。`SHG` 借作後字韻母 `INE`。

#### 音序簡拼法

將詞語的音序簡拼組合成一個並擊碼。

  - `SK` 略代「可是」。
  - `SB` 略代「不是」。
  - `SHB` 略代「是不是」。

  - `SHG` 略代「什麼」。
    將 `S/FB` 優化爲同一行的三鍵並擊。

「什麼」的縮略碼可衍生出常用短語：

  - `SHGIN`「什麼人」
  - `SHGR`「什麼事」
  - `SHGO`「什麼時候」
  - `SHGIR`「什麼意思」
  - `SHGUARO`「什麼情況」

其他無理編碼：

  - `ZFB`「沒有」
  - `ZFBIR`「有沒有」
  - `KR`「可以」
  - `CLDÜE`「越來越」
